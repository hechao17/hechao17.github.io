发表于 2018-02-08 | 分类于 学海无涯

bolg从github迁移到coding

配置共钥

以前配置过 github 的共钥，可以直接部署到 Coding 上。具体步骤就是将本地 C:\Users\Administrator\.ssh 目录下的 id_rsa.pub 的密钥写入到 Codding/我的账户/SSH共钥中，在 git bash 可以测试是否添加成功：

ssh -T git@git.coding.net --如果成功会返回下面的提示：

Coding 提示: Hello username, You've connected to Coding.net via SSH. This is a personal key.
username，你好，你已经通过 SSH 协议认证 Coding.net 服务，这是一个个人公钥

创建Coding Pages

coding pages 支持用户 pages 和 项目 pages，其中创建用户 pages 可以通过 {user_name}.coding.me 来访问； 创建项目 pages，则通过 {user_name}.coding.me/{project_name} 来访问，其中的 {user_name} 是你的 Coding 帐号名。

理解了coding pages的创建过程，就可以选择自己选择的创建类型，我们先创建一个 coding 项目，如果是创建用户 pages，项目名称就需要固定为 {user_name}.coding.me，创建后，修改本地 hexo 博客目录下的配置文件 _config.yml 中的两个配置，如下：

deploy:
  type: git
  repo: git@git.coding.net:{user_name}/{user_name}.coding.me.git

url: {user_name}.coding.me
root: /

设置完毕后，可以通过 git bash 的 hexo d 指令部署到 coding pages，然后就可以访问博客了。

Github和Coding共同部署

将 repo 同时加入两个对应仓库地址即可。

deploy:
  type: git
  repo: 
    coding: git@git.coding.net:{user_name}/{user_name}.coding.me.git
    github: git@github.com:{user_name}/{user_name}.github.io.git
