新的开始

发表于 2018-01-31 | 分类于 hexo

这是新的开始，我用hexo创建了第一篇文章，并在Gigt Page上部署。

环境的搭建

当然是需要在电脑上安装好hexo的环境，包括：

  1. 安装git
  2. node.js
  3. hexo

这些安装的方法随便在网上就能找到，就不详细叙述了。

创建本地bolg

在需要创建blog的文件夹中，启动git bash窗口。然后输入指令

hexo init 或者 hexo init (folder)

其中带 (folder) 是在你的指定的目录下在创建一个 folder 的目录。创建完成后，会自动成才一系列的默认的文件夹和文件。

然后就是编译，形成可以显示的网页文件，我们通过命令 hexo generate 或者 hexo g 来完成

编译完成后，如果需要本地查看效果，可以通过指令 hexo server 或者 hexo s 来启动本地服务器。打开浏览器 http://localhost:4000/ 就可以观察到自己的bolg效果了。

添加Blog内容

比如我要添加一篇新的博客，名字叫做新的开始，还是在git bash窗口中，通过指令 hexo new 新的开始 ，那么会在文件目录下的 source\_posts 下创建一个 新的开始.md 文件。我们编辑这个md文件就可以编辑Blog了。

md文件是markdown格式的文件。目前有多种编辑器支持markdown的编辑，比如 MarkdownPad 2

完成后，记得启动本地服务器，检查一下编辑后的效果。

部署到github

这里要求先熟悉 git 的操作。我们需要在 github 中先创建一个仓库，命名叫做 username.github.io，这样后续的操作会比较省心！！

上传之前， 先通过 hexo g 先编译产生html文件。上传到 github 的指令是 hexo deploy或者hexo d。

完成上传后，我们就可以通过网站访问博客了~~~，https://username.github.io/。

更换模版

hexo默认的是 landscape 模版。 太小白低端了，我们需要换一个更时尚的界面~~

据说hexo的Next模版不错，就去 https://github.com/iissnan/hexo-theme-next/releases 网站上去下载最新的Next模版；压缩文件解压到本地目录 themes 中，文件名修改为 next。

然后修改模版的配置文件， themes\next\_config.yml。 修改语言啦~， 图标啦~…细节大家从Next的官方文档中找寻把。

完成后，编译，重新部署到github中。
